Was ist Status?
Gibt an, ob eine Anfrage erfolgreich verarbeitet wurde
Statuscode:
1XX - Information
2XX - Erfolgreich
3XX - Umleitung
4XX - Client Fehler
5XX - Server Fehler
Client kann Status auslesen und verarbeiten

Was steht im Body?
Angefragte Ressource wird im Body verschickt
Abhängig von Status ist der Body optional 
200 —> Body enthält Daten
201 —> Kein Body nötig
204 —> Kein Body nötig
4XX —> Body kann Fehlermeldung beinhalten
5XX —> Body kann Fehlermeldung beinhalten

Zuordnung Methode - Statuscode
GET —> 200 (OK)
POST —> 201 (Created)
PUT —> 201 (Created / 204 (No Content)
PATCH —> 204 (No Content)
DELETE —> 200 (OK) / 204 (No Content)
